# 实验室样本交接与复核工作台

## 项目概述

本系统是一个实验室样本交接与复核工作台，支持前台收样员、检测员和质控负责人三种角色协同工作，实现样本从录入到复核的全流程管理。

## 技术栈

- **后端**: Node.js + Express + SQLite + JWT
- **前端**: Vue 3 + Vite + Element Plus
- **实时通信**: WebSocket

## 启动步骤

### 1. 安装后端依赖

```bash
cd backend
npm install
```

### 2. 初始化数据库

```bash
npm run init-db
```

这会创建 SQLite 数据库文件并插入示例数据。

### 3. 启动后端服务

```bash
npm start
```

后端服务将运行在 http://localhost:3000

### 4. 安装前端依赖

```bash
cd frontend
npm install
```

### 5. 启动前端开发服务器

```bash
npm run dev
```

前端页面将运行在 http://localhost:5173

## 功能说明

### 收样员
- 录入新样本（来源单位、样本类型、送检项目、接收时间）
- 查看待交接样本列表
- 批量分配样本给检测员
- 查看操作记录

### 检测员
- 接收分配给自己的样本
- 更新检测进度（未开始、进行中、已完成）
- 补充异常说明
- 提交样本进行复核
- 重新提交被退回的样本

### 质控负责人
- 查看待复核样本
- 处理样本（通过、退回补充说明、标记作废）
- 添加复核意见
- 查看所有状态的样本记录

## 核心特性

1. **并发安全**: 使用乐观锁（version字段）防止重复操作
2. **实时同步**: WebSocket实时推送状态变更，各页面自动刷新
3. **权限控制**: 后端中间件强制校验角色权限，非隐藏式控制
4. **审计日志**: 所有操作（创建、分配、接收、复核、作废）均有完整记录

## 数据库结构

- users: 用户表（含角色字段）
- samples: 样本表（含version乐观锁字段）
- sample_assignments: 样本分配记录
- inspection_records: 检测记录
- review_records: 复核记录
- audit_logs: 审计日志
